    - Test: Degenerate centroid detection -- after training on 10K vectors with m=8 k=256, no two centroids in same subspace closer than 1e-6 L2
    - Test: Parallel subspace training (behind persistence feature) produces same-quality codebooks as sequential
    - Test: ProductQuantizer with rotation=None serializes/deserializes correctly via postcard
    - Test: recall@10 >= 85% on 1000 clustered 64-dim vectors with m=8 k=16

    1. Add `ndarray` to Cargo.toml as optional dep gated by `persistence` feature:
       ```toml
       [dependencies.ndarray]
       version = "0.16"
       default-features = false
       optional = true
       ```
       Update persistence feature: `persistence = ["dep:memmap2", "dep:rayon", "dep:tokio", "dep:ndarray"]`

    2. Extend `ProductQuantizer` struct with rotation field:
       ```rust
       pub struct ProductQuantizer {
           pub codebook: PQCodebook,
           /// OPQ rotation matrix (flattened row-major D x D). None if OPQ disabled.
           pub rotation: Option<Vec<f32>>,
       }
       ```
       Update all constructors to set `rotation: None`.

    3. Modify `kmeans_train` function:
       - Change `max_iters` parameter from 25 to 50 in the call from `ProductQuantizer::train`
       - Add convergence check: compute max relative centroid movement after each update step. Break if `max_delta < 0.01` (1% threshold).
       - Replace empty cluster re-seeding: instead of `samples[cluster % samples.len()]`, split the largest cluster (clone its centroid + add small random perturbation via `rng.gen::<f32>() * 1e-4`). Track `largest_cluster` via `counts.iter().enumerate().max_by_key()`.
       - Keep `rng` as `rand::thread_rng()` for the perturbation.

    4. Add post-training degenerate centroid detection in `ProductQuantizer::train`:
       After building centroids, iterate over each subspace and check all centroid pairs within that subspace. If any pair has L2 distance < 1e-6, log `tracing::warn!("degenerate centroids detected in subspace {subspace}: centroids {i} and {j} distance {dist}")`. Do NOT fail -- just warn.

    5. Add parallel subspace training behind persistence feature gate:
       ```rust
       #[cfg(feature = "persistence")]
       {
           use rayon::prelude::*;
           centroids = (0..num_subspaces)
               .into_par_iter()
               .map(|subspace| {
                   let start = subspace * subspace_dim;
                   let end = start + subspace_dim;
                   let sub_vectors: Vec<Vec<f32>> = vectors.iter().map(|v| v[start..end].to_vec()).collect();
                   kmeans_train(&sub_vectors, num_centroids, 50)
               })
               .collect();
       }
       #[cfg(not(feature = "persistence"))]
       {
           // existing sequential loop
       }
       ```

    6. Add a test helper function `generate_clustered_vectors(n: usize, dim: usize, num_clusters: usize, seed: u64) -> Vec<Vec<f32>>` that generates vectors grouped in distinct clusters using seeded RNG. Place in the `#[cfg(test)] mod tests` block.
